## 1. Цель работы
Исследовать эффективность базового перебора с возвратом и его оптимизаций (эвристики, отсечения, интеллектуальный возврат) на двух задачах: гамильтонов путь по сетке и 15-пазл.

## 2. Постановка задачи

### 2.1. Задача о пути по сетке `N×M`
Найти путь из стартовой клетки в финишную, проходящий через каждую клетку ровно один раз.

Базовый подход: полный перебор с возвратом.  
Модификации:
1. Эвристика Варнсдорфа.
2. Отсечение по связности.
3. Backjumping.

### 2.2. Задача 15-пазла
Для заданной стартовой конфигурации найти последовательность ходов, переводящую поле в эталонное состояние.

Базовый эвристический подход: оценка расстояния до цели.  
Модификации:
1. BFS.
2. IDA*.
3. Backjumping.

## 3. Теоретическая часть

### 3.1. Полный перебор с возвратом
Дерево поиска строится рекурсивно; при невозможности продолжения выполняется возврат.

### 3.2. Отсечения и эвристики
- **Варнсдорф:** сначала переход в вершины с минимальным числом дальнейших ходов.
- **Проверка связности:** отбрасывание частичных путей, которые разрывают граф оставшихся клеток.
- **Backjumping:** возврат не на предыдущий уровень, а на конфликтный уровень.

### 3.3. Эвристики для 15-пазла
Используется оценка:
\[
h(n)=h_{Manhattan}(n)+h_{LinearConflict}(n)
\]
где:
- `h_Manhattan` — сумма манхэттенских расстояний плиток,
- `h_LinearConflict` — штраф за конфликтующие плитки в строках/столбцах.

Для A*/IDA* это ускоряет поиск, сохраняя информативность эвристики.

## 4. Формализация и спецификация программы
- Вход: параметры сетки/пазла, старт/финиш, лимиты глубины, лимиты времени, выбор алгоритмов.
- Выход: признак нахождения решения, длина решения, время, число операций, комментарий.
- Интерфейс: PyQt5, табы для двух задач, таблицы сравнения и предпросмотр шагов решения.

## 5. Выполненные улучшения реализации
1. Вычисления вынесены из UI-потока в фоновый поток, чтобы окно не «зависало».
2. Алгоритмы запускаются параллельно через `ProcessPoolExecutor(max_workers=4)` — загрузка до 4 ядер.
3. Для 15-пазла добавлена более сильная эвристика (Manhattan + Linear Conflict).
4. Интерфейс обновлён:
   - единый стиль вкладок и контролов;
   - укрупнённые элементы управления и увеличенная рабочая область;
   - статус выполнения («идёт вычисление», «завершено», «ошибка»);
   - блокировка кнопки запуска во время расчёта.
5. UI-код разделён на три части:
   - `main.py` (окно приложения и общая стилизация),
   - `hamiltonian_tab.py` (задача сетки `N×M`),
   - `puzzle_tab.py` (задача 15-пазла).
6. Для A* добавлена каскадная стратегия (оптимальный A* + ускоренные Weighted A* стадии), что повышает шанс быстро найти решение в заданном тайм-лимите.

## 6. Контрольный пример

### 6.1. Сетка
1. Размер `7×7`.
2. Старт и финиш задаются кликами.
3. Сравнение базового перебора и модификаций по времени/операциям.

### 6.2. 15-пазл
1. Генерация случайной разрешимой конфигурации.
2. Запуск A*, BFS, IDA*, Backjumping (по чекбоксам).
3. Сравнение длины решения, времени и операций.

## 7. Анализ результатов
- Основная причина «подвисания» — выполнение тяжёлых расчётов в UI-потоке.
- После выноса вычислений интерфейс остаётся отзывчивым.
- Параллельный запуск на 4 процессах сокращает общее время сравнения набора алгоритмов.
- Усиленная эвристика в 15-пазле уменьшает пространство поиска и повышает вероятность найти решение в лимитах.

## 8. Вывод
Задачи полного перебора успешно решаются с практическими оптимизациями: эвристики, отсечения ветвей, backjumping и параллельные вычисления.  
Ключевой практический эффект — рост отзывчивости интерфейса и более стабильное получение решений в заданных лимитах.
