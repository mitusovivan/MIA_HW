# GPLA-12: cumulant features (K2–K6) + 3 tasks (pressure / noise / microphone)

Этот проект запускает Python‑скрипт, который строит признаки по **кумулянтному методу** (через центральные моменты) для акустических сигналов из датасета **GPLA‑12** и решает 3 отдельные задачи классификации:

1) **pressure**: давление `0.2 / 0.4 / 0.5 MPa`  
2) **noise**: `noiseless / noisy`  
3) **microphone**: `mic1 / mic2`

> Скрипт рассчитан на **GPLA‑12 v1**: `data.csv` (684×1460) + `label.csv` (684×1, метки 1..12).

## 5. Настройки в скрипте

Откройте файл `gpla12_cumulants_3tasks_easy_run.py` и найдите блок:

```python
# USER SETTINGS (edit these if needed)
MODE = "nofilter"   # "nofilter" or "bands"
WIN_LEN = 256
HOP = 128
TEST_SIZE = 0.25
SEED = 42
OUT_FEATURES_CSV = "features_gpla12_cumulants_3tasks.csv"
```

### 5.1. Режимы признаков

#### MODE = "nofilter"
Кумулянты считаются прямо по сигналу (без фильтрации).

#### MODE = "bands"
Сигнал пропускается через несколько **полосовых фильтров** в нормированных частотах (0..Nyquist), и кумулянты считаются отдельно по каждой полосе.

Это сделано потому, что в статье GPLA‑12 не указана исходная частота дискретизации **Fs**, а данные уже “preprocessed” до длины 1460 за 5 секунд, поэтому стандартные полосы в кГц (например 5–10 кГц) напрямую применить нельзя.

---

## 6. Что именно считается (кумулянты)

Для каждого окна сигнала считаются центральные моменты `mu2..mu6`, затем кумулянты:

- `K2 = mu2`
- `K3 = mu3`
- `K4 = mu4 - 3*mu2^2`
- `K5 = mu5 - 10*mu3*mu2`
- `K6 = mu6 - 15*mu4*mu2 - 10*mu3^2 + 30*mu2^3`

Далее по всем окнам берутся агрегаты:
- `mean(K2..K6)`
- `std(K2..K6)`

Итого:
- в режиме `nofilter`: 10 признаков на сигнал
- в режиме `bands`: 10 × (число полос) признаков на сигнал

---

## 7. Маппинг исходных меток 1..12 в 3 задачи

По таблице классов GPLA‑12:

- `mic`:
  - 1..6 → mic1 (0)
  - 7..12 → mic2 (1)

- `noise`:
  - нечётные метки → noiseless (0)
  - чётные метки → noisy (1)

- `pressure`:
  - (1,2) и (7,8) → 0.2 MPa (0)
  - (3,4) и (9,10) → 0.4 MPa (1)
  - (5,6) и (11,12) → 0.5 MPa (2)

## 8. Результаты и интерпретация (подсказка)

Обычно:
- задача `microphone` решается очень хорошо (кумулянты чувствительны к тракту/датчику),
- задача `noise` тоже хорошо отделяется,
- `pressure` сложнее (эффект тоньше и часто требуется более богатый набор признаков).

## 9. Если что-то не запускается

1) Проверьте, что `data.csv` и `label.csv` реально лежат рядом со скриптом и так названы.
2) Проверьте, что Python видит пакеты:
   ```bash
   python -c "import numpy,pandas,scipy,sklearn; print('ok')"
   ```
3) Если появятся ошибки фильтрации в режиме `bands` — можно временно перейти на `MODE="nofilter"` или подстроить границы полос в коде.
